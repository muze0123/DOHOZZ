## 二、功能概述

### 2.1 背景
达人公海功能管理已释放到公海的达人资源，支持多维度筛选、批量跟进、分配、删除等操作，帮助企业高效利用公海达人资源。

### 2.2 目标
- 展示公海达人列表及详细信息
- 支持多维度筛选（平台、国家、分类、时间等）
- 支持批量操作（跟进、分配、删除）
- 支持查看达人标签和释放时间
- 提供达人数据指标展示

### 2.3 范围
- **包含**：达人公海列表展示、筛选区、批量操作、单个操作、分页功能
- **不包含**：达人详情页、导出功能、标签管理

### 4.2 筛选区结构（三行）

**第一行：国家地区**
- 标签：「国家地区」
- 默认按钮：「全部」
- 选项标签：「美国」「马来西亚」等（带删除叉号）
- 展开/收起按钮

**第二行：带货分类**
- 标签：「带货分类」
- 默认按钮：「全部」
- 选项标签：「美妆个护」「鲜花绿植」等

**第三行：带货信息**
- 标签：「带货信息」
- 下拉筛选器：「带货水平」「平均互动率」（带问号图标）
- 筛选标签按钮

### 4.3 收起状态
- 只保留：平台Tab
- 隐藏：搜索框、筛选区

## 六、批量操作规范

### 6.1 批量操作下拉菜单

触发：点击「批量操作」按钮
- 批量跟进：#333333 文字
- 批量分配：#333333 文字
- 批量删除：#FF0000 文字

### 6.2 批量分配弹窗（el-dialog）

**结构**：
- 标题：「批量分配达人」
- 内容：已选 X 位达人提示 + BD 下拉选择框
- 底部：取消按钮 + 确定按钮

**交互**：
- 未选中达人时，批量操作按钮禁用
- 未选择 BD 时，点击确定显示警告 Toast

### 6.3 批量跟进/删除弹窗

类似批量分配弹窗结构。

## 十、边界条件

| 场景 | 处理方式 |
|------|---------|
| 筛选后无数据 | 显示空状态「暂无数据」，筛选项保留 |
| 数据加载中 | 显示骨架屏 |
| 网络异常 | Toast 提示「网络异常，请稍后重试」 |
| 未选中达人点击批量操作 | 按钮禁用（灰色） |
| 批量分配未选BD | Toast 警告「请选择分配BD」 |
| 达人头像为空 | 显示灰色默认头像 |
| 达人名称过长 | 超过140px显示省略号 |

## 十一、实现参考

### 11.1 组件参考
- 参考页面：`NaturalOrderInfluencer.vue`、`BrandDiscovery.vue`
- UI 框架：Element Plus
- 表格组件：`el-table`
- 弹窗组件：`el-dialog`
- 下拉组件：`el-select`
- 分页组件：`el-pagination`

### 11.2 状态管理
- 使用 Pinia store 或组件内 `ref/reactive` 管理状态
- 筛选状态：`isFilterExpanded`（收起/展开）
- 选中状态：`selectedRows`（批量选中）
- 分页状态：`currentPage`、`pageSize`

### 11.3 排序实现
- 默认排序：`recentReleaseTime` 降序
- 支持排序字段：粉丝数、近30日成交金额、近30日销量、场均成交金额、最近释放时间
